# Argument Passing

`process_exec()`에서 사용자 프로그램을 위한 인자를 설정합니다.

## x86-64 Calling Convention

이 절은 Unix의 64비트 x86-64 구현에서 일반 함수 호출에 사용되는 convention 중 중요한 부분을 요약합니다. 간결함을 위해 일부 세부 사항은 생략되어 있습니다. 더 자세한 내용은 System V AMD64 ABI를 참고할 수 있습니다.

Calling convention은 다음과 같이 동작합니다.

- 사용자 수준 애플리케이션은 정수 인자를 전달할 때 `%rdi`, `%rsi`, `%rdx`, `%rcx`, `%r8`, `%r9` 순서의 레지스터를 사용합니다.
- 호출자는 다음에 실행할 명령어의 주소, 즉 return address를 스택에 push한 뒤 callee의 첫 번째 명령어로 jump합니다. x86-64의 `CALL` 명령어 하나가 이 두 작업을 모두 수행합니다.
- callee가 실행됩니다.
- callee에 반환값이 있으면 그 값을 `RAX` 레지스터에 저장합니다.
- callee는 스택에서 return address를 pop하고, x86-64의 `RET` 명령어를 사용해 그 주소가 가리키는 위치로 jump하여 반환합니다.

세 개의 `int` 인자를 받는 함수 `f()`를 생각해 봅시다. `f(1, 2, 3)`으로 호출되었다고 가정하면, 위의 3단계가 시작될 때 callee가 보는 sample stack frame과 register state는 다음과 같습니다. 초기 스택 주소는 임의입니다.

```text
                             +----------------+
stack pointer --> 0x4747fe70 | return address |
                             +----------------+
RDI: 0x0000000000000001 | RSI: 0x0000000000000002 | RDX: 0x0000000000000003
```

## Program Startup Details

사용자 프로그램을 위한 Pintos C library는 `lib/user/entry.c`의 `_start()`를 사용자 프로그램의 entry point로 지정합니다. 이 함수는 `main()`을 감싸는 wrapper이며, `main()`이 반환하면 `exit()`를 호출합니다.

```c
void
_start (int argc, char *argv[]) {
    exit (main (argc, argv));
}
```

커널은 사용자 프로그램이 실행을 시작하도록 허용하기 전에 initial function의 인자를 레지스터에 넣어야 합니다. 인자들은 일반 calling convention과 같은 방식으로 전달됩니다.

다음 예시 명령을 어떻게 처리할지 생각해 봅시다.

```text
/bin/ls -l foo bar
```

먼저 명령을 단어로 나눕니다.

```text
/bin/ls, -l, foo, bar
```

그다음 단어들을 스택의 top에 배치합니다. 단어들은 포인터를 통해 참조될 것이므로 순서는 중요하지 않습니다.

각 문자열의 주소와 null pointer sentinel을 right-to-left 순서로 스택에 push합니다. 이것들이 `argv`의 원소입니다. null pointer sentinel은 C standard가 요구하는 것처럼 `argv[argc]`가 null pointer가 되도록 보장합니다. 이 순서는 `argv[0]`이 가장 낮은 가상 주소에 위치하도록 합니다. Word-aligned access는 unaligned access보다 빠르므로, 최상의 성능을 위해 첫 번째 push 전에 stack pointer를 8의 배수로 내림 정렬합니다.

`%rsi`가 `argv`, 즉 `argv[0]`의 주소를 가리키도록 하고, `%rdi`를 `argc`로 설정합니다.

마지막으로 가짜 `return address`를 push합니다. entry function은 절대 return하지 않지만, 그 stack frame은 다른 함수와 같은 구조를 가져야 합니다.

아래 표는 사용자 프로그램이 시작되기 직전의 스택 상태와 관련 레지스터를 보여 줍니다. 스택은 아래 방향으로 성장한다는 점에 주의하세요.

| Address | Name | Data | Type |
| --- | --- | --- | --- |
| `0x4747fffc` | `argv[3][...]` | `'bar\0'` | `char[4]` |
| `0x4747fff8` | `argv[2][...]` | `'foo\0'` | `char[4]` |
| `0x4747fff5` | `argv[1][...]` | `'-l\0'` | `char[3]` |
| `0x4747ffed` | `argv[0][...]` | `'/bin/ls\0'` | `char[8]` |
| `0x4747ffe8` | `word-align` | `0` | `uint8_t[]` |
| `0x4747ffe0` | `argv[4]` | `0` | `char *` |
| `0x4747ffd8` | `argv[3]` | `0x4747fffc` | `char *` |
| `0x4747ffd0` | `argv[2]` | `0x4747fff8` | `char *` |
| `0x4747ffc8` | `argv[1]` | `0x4747fff5` | `char *` |
| `0x4747ffc0` | `argv[0]` | `0x4747ffed` | `char *` |
| `0x4747ffb8` | `return address` | `0` | `void (*) ()` |

```text
RDI: 4 | RSI: 0x4747ffc0
```

이 예시에서 stack pointer는 `0x4747ffb8`로 초기화됩니다. 위에서 보인 것처럼, 여러분의 코드는 `include/threads/vaddr.h`에 정의된 `USER_STACK`에서 스택을 시작해야 합니다.

인자 전달 코드를 디버깅할 때는 `<stdio.h>`에 선언된 non-standard 함수 `hex_dump()`가 유용할 수 있습니다.

## 구현해야 할 내용

Argument passing을 구현하세요.

현재 `process_exec()`는 새 프로세스에 인자를 전달하는 기능을 지원하지 않습니다. 이 기능을 구현하려면 `process_exec()`를 확장해서, 단순히 프로그램 파일 이름 하나를 인자로 받는 대신 공백을 기준으로 여러 단어로 나누도록 해야 합니다. 첫 번째 단어는 프로그램 이름이고, 두 번째 단어는 첫 번째 인자이며, 그다음도 같은 방식입니다. 즉, `process_exec("grep foo bar")`는 `grep`을 실행하면서 두 인자 `foo`와 `bar`를 전달해야 합니다.

하나의 command line 안에서 여러 개의 공백은 하나의 공백과 동일하게 취급됩니다. 따라서 `process_exec("grep foo bar")`는 원래 예시와 동일합니다. Command line arguments의 길이에 합리적인 제한을 둘 수 있습니다. 예를 들어, 하나의 page, 즉 4 kB 안에 들어가는 인자들로 제한할 수 있습니다. 참고로 `pintos` utility가 커널에 전달할 수 있는 command-line arguments에는 이와 별개의 128바이트 제한이 있습니다.

인자 문자열은 원하는 방식으로 parse할 수 있습니다. 막막하다면 `include/lib/string.h`에 prototype이 있고 `lib/string.c`에 자세한 주석과 함께 구현되어 있는 `strtok_r()`를 살펴보세요. 프롬프트에서 `man strtok_r`를 실행해 man page를 확인하면 더 자세히 알 수 있습니다.
