# 문서형 학습 리소스 관리 규칙

이 폴더는 PDF, 책, 노트, 강의자료 등 `문서형 학습 리소스`를 저장하고 정리하기 위한 공간이다.  
학습 md는 이 폴더의 리소스 인덱스를 기반으로 1차/2차 작업을 수행한다.

## 1) 목적

- 원문 자료(또는 추출본) 보관
- 1차 읽기(흐름 파악) 결과인 “모르는 항목 리스트” 수집
- 2차 정리 시 위치 기반 반영
- 최종적으로 `docs/HY`의 학습 문서와 상호 참조 관계 유지

## 2) 폴더 구조(권장)

- `resources/materials/`  
  - 리소스별 루트 폴더: `도서명_저자_연도/` 또는 `강의명_채널_날짜/`
    - `meta.md` : 리소스 메타데이터(출처, 범위, 난이도)
    - `toc.md` : 목차/챕터 인덱스
    - `pass1-notes.md` : 1차 읽기 모르는 항목 리스트(원문 위치 기반)
    - `pass2-questions.md` : 2차 학습 질문·확인 요청
    - `draft-notes.md` : 초안 정리본(내부 노트)
    - `snippets/` : 원문 인용/핵심 문장/페이지 캡처 메모
- `resources/templates/`  
  - 생성 템플릿(`resource-template.md`)

## 3) 리소스 처리 파이프라인

### 3.1 1차 읽기 (BFS 모드)
1. 리소스 등록(`meta.md`, `toc.md` 확인)
2. 빠른 스캔 진행(심화 탐색 대신 전체 흐름 파악)
3. 모르는 개념/구간은 `pass1-notes.md`에 `장 번호 / 단락 / 질문` 형태로 기록
4. 리소스 인덱스 갱신

### 3.2 2차 정리 반영
1. 요청된 모르는 항목 목록(`pass1-notes.md` 또는 별도 질문) 수신
2. 항목별 `출처 위치`를 기준으로 학습 md에서 적절한 위치 탐색
3. 중복 제거 후 보강 삽입 (필요하면 예시/시각자료 추가)
4. 학습 md 업데이트 시 본문 하단에 `출처 추적` 링크 추가

### 3.3 문서 간 연결 규칙
- 학습 문서 각 항목에 `출처:` 항목을 두어 리소스/챕터를 연결
- 리소스 폴더에 `관련 학습 md` 목록을 기록
- 반대로 학습 md의 하단 `참고 리소스`에서 `resources/...` 경로를 역추적 가능하도록 유지

## 4) 파일명 및 메타 규칙

- 리소스 폴더명: `주제-영문축약-연도-식별자` (공백 최소화)
- `pass1-notes.md` 항목 예시
  - `- [ch05-2] 캐시 일관성의 문제점: CPU 캐시와 메모리 불일치 가능성`
- `meta.md`에는 최소 항목으로 다음을 포함
  - title, source_type(pdf/book/slide), author, year, pages, focus, start_date, end_date
- `status`: pending / studying / reviewed / extracted

## 5) 품질 기준

- 원문 출처가 있으면 `pass1-notes.md`에 근거 위치(챕터/절/페이지) 기입
- 핵심 개념은 1차 리스트에서 바로 문서화하지 않고, 2차 정리 시 정돈된 형태로 반영
- 복잡한 개념은 “예시 우선” 원칙에 따라 보강
- 동일 항목은 `pass1-notes`에서 중복 제거하며 `merged` 상태로 변환

## 6) 템플릿

- 새 리소스 등록 시 기본 템플릿 사용: [resource-template.md](./templates/resource-template.md)
