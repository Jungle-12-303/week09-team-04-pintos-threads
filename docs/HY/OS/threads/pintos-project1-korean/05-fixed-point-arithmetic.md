# Project 1-4: 고정소수점 실수 연산

## 상위/관련 링크 (필수)
- 상위: [OS 인덱스](../README.md)
- 관련: [고급 스케줄러(MLFQS)](./04-advanced-scheduler.md)

## 배경

우선순위 계산식에는 `recent_cpu`, `load_avg` 같은 실수 값이 들어간다.  
커널에서는 부동소수점 연산을 사용하지 않으므로, 정수로 이를 흉내내야 한다.

Pintos는 정수의 하위 비트를 소수부로 간주하는 고정소수점 방식을 쓴다.

예: 17.14 형식  
- 부호 1비트 + 정수부 17비트 + 소수부 14비트
- 실수 x는 `x * 2^14` 형태로 정수로 저장
- 최대값 근사: `(2^31 - 1) / 2^14 ≈ 131,071.999`

## 변환

p.q 형식에서 `f = 2^q`일 때:
- 정수/실수 → 고정소수점: `* f`
- 고정소수점 → 정수(버림): `/ f`
- 반올림:
  - 양수: `(x + f/2) / f`
  - 음수: `(x - f/2) / f`

## 기본 연산 규칙

`x`, `y`는 고정소수점, `n`은 정수, `f=2^q`

- 덧셈: `x + y`, `x - y`
- 정수와 혼합:
  - `x + n` -> `x + n * f`
  - `x - n` -> `x - n * f`
- 정수 곱/나눗셈:
  - `x * n`, `x / n`

## 곱셈의 함정

두 고정소수점 곱은 스케일이 `f`만큼 한 번 더 늘어난다.

- 잘못: `x * y`
- 올바른 기본: `((int64_t)x) * y / f`

왜냐하면 소수점이 `q`비트만큼 왼쪽으로 지나치게 이동되었기 때문이다.

또한 오버플로우 방지를 위해 **반드시 64비트 중간 계산** 사용.

예시:
- `64`는 17.14에서 `1,048,576`이고 제곱은 `2^40`로 32비트 오버플로우.
- 64비트 캐스팅으로 안전 처리.

## 나눗셈의 함정

두 고정소수점 나눗셈은 반대로 소수점이 `q`비트만큼 오른쪽으로 밀린다.  
따라서 나누기 전 피제수를 `f`배 해준다.

- `x / y = ((int64_t)x) * f / y`

역시 64비트 연산이 안전.

## 구현 표(요약)

| 연산 | C 코드 |
|---|---|
| 정수→고정소수점 | `n * f` |
| 고정소수점→정수(버림) | `x / f` |
| 고정소수점→정수(반올림) | `x >= 0 ? (x + f/2) / f : (x - f/2) / f` |
| 고정소수점 덧셈 | `x + y` |
| 고정소수점 뺄셈 | `x - y` |
| 고정소수점+정수 덧셈 | `x + n * f` |
| 고정소수점-정수 | `x - n * f` |
| 고정소수점 곱 | `((int64_t) x) * y / f` |
| 고정소수점×정수 | `x * n` |
| 고정소수점 나눗셈 | `((int64_t) x) * f / y` |
| 고정소수점÷정수 | `x / n` |

## 실무 팁

- shift(`<<`, `>>`)보다 `*f`/`/f` 사용이 더 안전한 이유:
  - 연산자 우선순위 혼란 감소
  - 음수에서 shift 동작 차이 위험 감소
- 실수 계산은 모두 정수 기반이나, 반올림/스케일 보정이 핵심.
- 2개 고정소수점 곱은 항상 스케일 보정(1회 나눗셈) + 넓은 정수 타입을 기본 규칙으로 둔다.

## 과제 연동 함수

스레드 코드에 이미 제공된 스켈레톤에서 아래 함수를 구현해야 한다.

```c
int thread_get_recent_cpu (void);
```

- 현재 스레드 `recent_cpu`의 **100배 정수 반올림**값 반환

```c
int thread_get_load_avg (void);
```

- 현재 시스템 `load_avg`의 **100배 정수 반올림**값 반환
